# Clean Code by 로버트 C. 마틴

### 2장

이름은 아래 내용을 주석없이 모두 답해서 의도를 전달할 수 있어야한다.

- 변수의 존재 이유
- 수행 기능
- 사용법

주석이 있다면, 의도를 정확히 전달하지 못한 것이다.

그릇된 정보(리스트가 아닌데 리스트라 하기), 흡사한 이름, 무의미한 불용어(a, the, ...) 등을 사용하지 말자

명확한 이름을  사용하자 

- getActiveAccount(), getActiveAccounts(), getActiveAccountInfo() 처럼 서로 비슷하거나 애매한 이름은 혼란을 가져온다

검색하기 쉬운 이름을 사용하자

인코딩( 변수에 코딩을 넣는 것)을 피하자

- 타입을 넣거나 접두사로 의미부여 등을 무의미하게 하지 말자

클래스 - 명사, 메서드 - 동사가 좋다

기발하거나, 은어의 사용은 피해야한다.

한 개념에 한 단어만 사용해라

- 추상 개념 하나당 하나의 단어를 선택하고 통일성 있게 사용해야한다
- get, tetrieve, fetch 등 뜻은 비슷하지만 '가져온다'의 개념에 매번 다르게 매핑하면 매 번 큰 혼란을 가져온다.
- 하지만, 반대로 비슷하다고 다른 개념을 같이 매핑시켜도 안된다. `add`를 더하기로 사용했고 그 개념을 확장시켜서 list에 insert함수를 `add`로 사용하서는 안된다. 이것은 insert 함수다.

기술 영역 용어와 문제 영역 용어를 이용해서 이름을 지어라 ( 기술 > 용어)

의미 있는 맥락을 추가해라

- city, state, firstName 보다 addrCity, addrState , addrFirstName 등 으로 바꾸어 더욱 구체적으로 변수가 무슨 변수인지 설명해라
- 이는 함수에도 적용된다. 코드예시는 생략한다.
- 반대로 불필요한 맥락은 추가하지 마라. 이는 읽기 힘들게 할 뿐이다.
