# Clean Code by 로버트 C. 마틴

### 1장

Code는 항상 존재할 것이며, 우리는 깨끗한 코드를 짜야한다

메모리누수, 경쟁상태, 일관성없는 명명법을 조심하자.

좋은 코드란 무엇일까?

- 비야네 스트롭스트룹 : 한 가지에 집중해서 깔끔하게 코드를 작성하자

- 그래디 부치 , 데이브 토마스 : 가독성이 높아야한다

- 마이클 페더스 : 세세한 부분까지 주의를 기울인 코드

- 론 제프리스 : 이름을 잘 짓고(표현), 중복을 없애라

- 위드 커닝햄 : 코드가 그 문제를 풀기 위한 언어로 보인다면 잘 짠 것이다

  - 즉, 코드가 군더더기 없이 깔끔해야한다.

- 저자 : 읽기 쉬운 코드

이 책은 정답이 아니다. 저자가 찾은 가장 깔끔한 코드를 짜는 법이다.

코드를 유지보수 할 때, 반드시 이전보다 보기 좋은 코드를 만들어라

완독 후 저자의 PPP 책을 읽길 권장 SRP, OCP, DIP 등 다양한 설계 원칙을 다룸.

### 모르는 용어들

TDD : Test-Driven Development 테스트 주도 개발

휴리스틱 : 경험 기반 선택, 알고리즘과 반대 선택 기법

르블랑의 법칙(Leblanc’s Law) :나중은 절대 돌아오지 않는다는 법칙입니다.나쁜 코드를 구현하면 나중은 오지 않지 않습니다.

OCP : open closed principle, 확장에는 열려있으나 변경에는 닫혀있어야한다.
