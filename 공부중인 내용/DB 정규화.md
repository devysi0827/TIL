# DB 정규화

1. 정규화란?
   - 하나의 릴레이션에 하나의 의미만 존재하도록 릴레이션을 분해하는 과정이며, 데이터의 일관성, 최소한의 데이터 중복, 최대한의 데이터 유연성을 위한 방법입니다. 
2.  정규화를 하는 이유

- 불필요한 데이터 (data redundancy)를 제거해 불필요한 중복을 최소화하기 위해서
- 삽입/갱신/삭제 시 발생할 수 있는 각종 이상 현상(Anomaly)을 방지하기 위해서

- cf). 주목적은 아니지만, DB 구조 확장 시 재디자인을 최소화, 다양한 관점에서의 query를 지원하기 위해서
