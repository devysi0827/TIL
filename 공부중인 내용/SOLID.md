# SOLID

### 1. SRP(Single Responsibility Principle)

- 작성된 클래스는 하나의 기능만 가지며, 클래스가 제공하는 모든 서비스는 그 하나의 책임을 수행하는데 집중되어 있어야한다는 원칙입니다.
- 이는 어떤 변화에 대해서 클래스를 변경해야하는 이유가 오직 하나여야한다는 것입니다.
- 코드 가독성/ 유지보수의 용이성에 장점이 있으나 실무에서 실제로 적용하기 어려워 연습이 필요합니다.



### 2. OCP(Open Close Principle)

- 확장에는 열려있고 변경에는 닫혀있어야 한다는 원리입니다.
- 설계 시, 변경 될 것과 변경되면 안 될 것을 확실히 구분해야하며 인터페이스를 변경하지 않고 인터페이스에 맞게 설계해야함. 이는 결국 추상화를 얼마나 잘 하는 지에 따라서도 결정될 수 있음.
