
# React Rendering

### 목적

- 리액트는 언제 컴포넌트를 렌더링하고, 언제 업데이트를 하여 리렌더링을 시키는 지 알아보자.



### 시기

- 업데이트가 예정된 상태(state)가 있을 경우
- 부모 컴포넌트가 렌더링 되었을 때, 아래 조건들 중 하나라도 충족할 경우
  - 컴포넌트가 이전에 렌더링 되었어야함, 마운트 되었어야함
  - 변경된 props(참조)가 없어야함
  - 컴포넌트 내 context 값이 변경되지 않아야함
  - 컴포넌트 내 예정된 상태 업데이트가 없어야함



### 렌더링이란?

- `리액트가 컴포넌트를 렌더링한다`

  => 리액트가 새로운 컴포넌트를 호출하여서 기존의 컴포넌트를 대체하는 방식이다.

- 또한, 컴포넌트가 만든 하나의 상태 업데이트가 반드시 하나의 렌더링으로 변환되지 않습니다.

  - 리액트는 의미있는 상태변화라 생각하지 못할 수 있습니다.(object.is가 판단)
  - 리액트는 상태 업데이트를 하나의 렌더 패스로 일괄 처리를 진행하려합니다.(리액트 18부터 완전히 가능)

- 업데이트란?

  - 리액트에게 컴포넌트의 상태를 변경하고 UI에 반영하도록 요청하는 작업 :
    - 즉, 리액트가 컴포넌트를 호출(렌더링)하는 이유
    - 이 때, 이 상태변화를 반영할 지, 몇 번 렌더링 할 것인지, 걸린 시간은 다양한 요인에 따라서 달라집니다.
    - ''리액트가 UI를 업데이트한다"는 기존의 DOM 노드를 변환하거나 새롭게 생성한다는 의미입니다.



### 리액트의 렌더링 유형

- 능동적인 렌더링
  - 컴포넌트 상태 변화를 예약합니다
  - ReactDOM.render를 호출합니다
- 수동적인 렌더링
  - 부모 컴포넌트의 렌더링에 따라서, 같이 렌더링이 됩니다.(예외 조건을 하나라도 만족시킬 경우)
