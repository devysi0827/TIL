**WEB**

- http vs https
  - http는 단순 전송방식으로, 암호화가 안되어 있어서 다른 유저도 쉽게 응답 기록을 확인할 수 있음
  - https는 비대칭키 전송방식을 이용하여 암호화/복호화 된 자료를 줌
- cors
  - sop 정책에 따라서, cors 정책에 따르는 항목만 교차 출처에 대해서 정보를 받아올 수 있음
  - 서버 간 통신 or access-controll-allow-origin 등의 헤더 설정을 함
  - csrf => 사용자가 웹 사이트에 자기도 모르는 사이 해커가 정한 동작을 수행함(비번을 해커가 원하는 대로 변경하는 로직 실행)
  - xss => 사용자가 이미 로그인되어 있는 점을 이용하여, 사용자인척 서버에 요청을 보냄 (백만원을 00로 송금해란 로직 실행)
- spa
- hash
- ssr VS csr VS ssg
  - ssr : 서버에서 렌더링이 되는 상태로 html을 제공
  - csr : 클라인언트가 다 다운받아서 렌더링을 진행
  - ssg : 페이지를 html로 다 만들어서 html만 다운
